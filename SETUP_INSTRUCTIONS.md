# Установка и запуск Jarvis

Тут собрал всё, что нужно знать, чтобы поднять помощника у себя README — это
короткий старт, а здесь подробности: настройки, локальные модели, микрофон,
безопасность и телеграм

## Python и зависимости

У меня всё крутится на системном Python 3.10:

```text
%LOCALAPPDATA%\Programs\Python\Python310\python.exe
```

Ставить библиотеки надо именно в него, иначе часть аудио- и CUDA-пакетов не
подтянется:

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python310\python.exe" -m pip install -r requirements.txt
```

Python 3.10 официально доживает до 4 октября 2026 года Переезжать на версию
поновее лучше отдельно и не второпях — заодно проверить звук, CUDA, окно
pywebview и тесты

## Настройки

Копируешь `jarvis_config.example.json` в `jarvis_config.json` и вписываешь ключ
OpenRouter Настоящий файл гит игнорирует Переменные окружения, если они заданы,
перебивают то, что в JSON

Основное, что там есть:

```jsonc
{
  "OPENROUTER_API_KEY": "sk-or-v1-...",
  "JARVIS_LLM": "local",
  "OLLAMA_MODEL": "qwen2.5:3b",
  "JARVIS_LLM_DEADLINE": "1.5",
  "STT_ENGINE": "whisper",
  "WHISPER_MODEL": "small",
  "JARVIS_PAUSE_THRESHOLD": "2.6",
  "JARVIS_WAKE_COMMAND_WINDOW": "10.0",
  "JARVIS_PHRASE_TIME_LIMIT": "45.0",
  "TTS_ENGINE": "auto",
  "PIPER_VOICE": "dmitri",
  "JARVIS_FOLLOWUP_MODE": "strict",
  "JARVIS_UI": "on",
  "JARVIS_OVERLAY": "on"
}
```

Пара пояснений, чтобы не гадать:

- `STT_ENGINE=whisper` — распознавание локально через faster-whisper, а если
  что-то пойдёт не так, само переключится на Google
- `WHISPER_MODEL=small` — брать именно `small` С `base` он путает слово "Джарвис"
  (слышит "Жар весь") и кажется, будто помощник оглох
- `TTS_ENGINE=auto` — если есть модель Piper, говорит через неё, иначе через
  edge-tts от Microsoft
- `TTS_ENGINE=edge` — всегда голос Microsoft (DmitryNeural)
- `TTS_ENGINE=piper` — локальный голос из папки `piper_models/`
- `TTS_ENGINE=xtts` — тяжёлое клонирование голоса, нужно редко

Ещё настройки можно менять прямо в окне по кнопке ⚙ Сохраняются они туда же, в
`jarvis_config.json`, и применяются после перезапуска

## Ollama (если хочется быстро и без интернета)

Ставишь Ollama и качаешь модель:

```powershell
ollama pull qwen2.5:3b
```

Jarvis сам проверит, запущен ли сервер, при необходимости поднимет `ollama serve`,
прогреет модель, а тяжёлые запросы отдаст в облачный DeepSeek

## Запуск

Обычный способ — с видимым окном консоли, чтобы видеть логи:

```powershell
python jarvis.py
```

Подробности пишутся в `logs/jarvis_YYYY-MM-DD.log`

Если хочется ярлык на рабочем столе, который запускает всё без чёрного окна
консоли (через `pythonw.exe`):

```powershell
powershell -ExecutionPolicy Bypass -File .\install_shortcut.ps1
```

## Микрофон и голос

- Если микрофонов несколько и берётся не тот — задай `JARVIS_MIC_INDEX` номером из
  списка, который печатается при старте
- `JARVIS_PAUSE_THRESHOLD` — сколько тишины считается концом фразы По умолчанию
  `2.6`, чтобы можно было задуматься посреди длинного вопроса и он не перебивал
- Если просто позвать "Джарвис" и замолчать, откроется тихое окно на 10 секунд —
  он не будет говорить "Слушаю" поверх начала твоей следующей фразы
- `JARVIS_PHRASE_TIME_LIMIT=45` — чтобы можно было диктовать длинное, вплоть до кода
- После ответа 15 секунд можно продолжать без слова "Джарвис"
- Голос Piper выбирается через `PIPER_VOICE`: `dmitri`, `ruslan`, `denis`, `irina`

## Про безопасность

Когда помощник выполняет код или команды, он не даст снести систему: блокируются
форматирование дисков, удаление Windows, системного реестра, самого проекта и базы
Obsidian Всё остальное (обычная автоматизация, файлы вне защищённых папок)
выполняется сразу, без лишних подтверждений

## Проверка после правок

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python310\python.exe" test_regression.py
& "$env:LOCALAPPDATA\Programs\Python\Python310\python.exe" test_jarvis_functions.py
& "$env:LOCALAPPDATA\Programs\Python\Python310\python.exe" test_safety.py
& "$env:LOCALAPPDATA\Programs\Python\Python310\python.exe" health_check.py
```

Два момента, на которые я сам пару раз наступал: не перезапускай фоновый слушатель
внутри основного цикла и не убирай отдельный поток из логики таймаутов LLM — на оба
случая есть тесты, которые сразу краснеют

## Телеграм (личный аккаунт)

Для телеграма используется официальный клиент Telethon Файлы `Telegram Desktop/tdata`
он не трогает и никуда не копирует

1. Заходишь на `https://my.telegram.org/apps` и создаёшь приложение для своего аккаунта
2. В настройках Jarvis вписываешь `API ID`, `API Hash` и телефон в виде `+79991234567`
3. Жмёшь "получить код", вводишь код из телеграма и "подключить"
4. Если стоит двухэтапная проверка — вводишь пароль 2FA Он никуда не сохраняется

Сессия лежит локально в `telegram_data/` и в гит не уходит Выгруженные диалоги
падают в `Documents/Jarvis Telegram Exports/` обычным Markdown

Голосом это выглядит так:

- "Джарвис, покажи мои чаты в телеграме"
- "Джарвис, прочитай последние 10 сообщений из чата с Иваном"
- "Джарвис, выгрузи из телеграма диалог с Иваном, последние 200 сообщений"
- "Джарвис, отправь Ивану в телеграме сообщение буду через час" — отправит только
  после отдельного подтверждения
